# Development Workflow

A lightweight workflow tailored for this repo.

## 1. Initialize
- Read `README.md`, `CONTRIBUTING.md`, `CHANGELOG.md`, and `TODO.md` for context.
- Check `git status` and confirm the active branch.

## 2. Plan
- Summarize the objective and relevant files.
- Identify risks and decide on verification steps.

## 3. Implement
- Make focused changes in dependency order.
- Keep edits minimal and consistent with existing patterns.

## 4. Verify
- Run targeted checks when feasible (see `AGENTS.md`).
- Document any limitations that block running tests.

## 5. Deliver
- Write a Conventional Commit message.
- Prepare a PR summary and testing notes.
- Update memory artifacts in `.agent/context/` and `.agent/sessions/`.
